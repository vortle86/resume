Common HTML Form Elements

Forms are contained by <form> and </form> tags.
<form id="form" action="" method="post">
When you make a form, you need the following attributes inside the form tag: id="" action="" (with the address to send the info to) and method="" (almost always post.)
You might also need invisible pieces that identify the form to the form parser.
The most common type of form field is for text. It looks like this:
<input type="text" name="text">
The input tag is special because it doesn't require a closing tag.
Right now, this input field doesn't give any clue what goes in it. To fix this, you can use a <label> tag:
<label for="favChar">Put your favorite Deadwood character here!</label>
<input type="text" name="favChar" id="favChar">
When you use the labeltag, make sure the for attribute of the label matches the id attribute of the input. Using labels is also a helpful accessibility practice for people who use screenreaders to access your form.
You can also provide a hint by using the value attribute to show prompt text inside the field:
<label for="favChar">Put your favorite Deadwood character here!</label>
<input type="text" name="favChar" id="favChar" value="Favorite Deadwood Character">
But Woe! Our cue has been cut off by the size of the box. We need to change the size with the size attribute.
<label for="favChar">Put your favorite Deadwood character here!</label>
<input type="text" name="favChar" id="favChar" value="Favorite Deadwood Character" size=30>
If we want to weed out any degenerates who might have tried to pick Francis Wolcott (15 characters) while still allowing for the choice of Al Swearengen (13 characters) we can set a maximum size for our field (rejecting inputs beyond a certain length.)
<label for="favChar">Put your favorite Deadwood character here!</label>
<input type="text" name="favChar" id="favChar" value="Favorite Deadwood Character" maxsize=14>
This size isn't long enough to hold our prompt, but it will keep out the scary serial killer fans so I'm ok with that.
If we want to still see our prompt, but continue weeding out the lunatics, we can set a size and maxsize.
<label for="favChar">Put your favorite Deadwood character here!</label>
<input type="text" name="favChar" id="favChar" value="Favorite Deadwood Character" size=30 maxsize=14>
There are also other kinds of input fields, including:
  • email (for an email address)
  • number (for a number)
  • tel (for a phone number)
  • url (caught the pattern yet?)
There are also lots of specialized input types, some of which we will show below, but others you'll have to explore yourself.
Sometimes we want an input, but don't want to give too much choice. Perhaps we want the user to be able to select only a single option from a pretty short list.
If we want to know whom the user regards as the protagonist of Deadwood, we might use a radio button, because we have a narrow view of the literary concept and have defined protagonist as being a single person role.
Who is the protagonist of Deadwood?
<label for="bullock">Seth Bullock</label>
<input type="radio" name="prot" id= "bullock" value="bullock">
<label for="swearengen">Al Swearengen</label>
<input type="radio" name="prot" id="swearengen" value="swearengen">
Notice that this time, we must have a name, which will be the same for all of our choices to tell the form parser what the information means, and a value to tell what the user chose.
Or perhaps you would like to allow the user to select more than one option. You would use checkboxes.
Which seasons of Deadwood have you watched?
Note that if you do not select "3" You will receive a special visit from a counselor who will help you explore the error of your ways and correct this problem in your life.
<input type="checkbox" name="zero" value=0> 0
1 2 3
Since we're pretty sure that all decent people will select "3" we could have it selected as a default.
<input type="checkbox" name="3" checked>
1 2 3
What if we want people to pick from a list, but don't want to use checkboxes or radio buttons? Selection fields to the rescue! For a single choice in a drop down menu,
Which is the best female character in Deadwood?
<select name="female">
<option value="alma">Alma Garrett-Ellsworth</option>
<option value="jane">Jane Canary</option>
<option value="jewel">Jewel</option>
<option value="trixie">Trixie</option>
<option value="martha">Martha Bullock</option>
</select>
Notice that both the select and the option tag require an element and a closing tag.
Now, perhaps we need to allow for multiple selections in a list, but we've got a serious vendetta against checkboxes.
Choose the lackeys who work for Al Swearengen
<select name="lackey" multiple="multiple">
<option value="dority">Dan Dority</option>
<option value="eb">E.B. Farnam</option>
<option value="richardson">Richardson</option>
<option value="johnny">Johnny Burns</option>
<option value="adams">Silas Adams</option>
<option value="leon">Leon</option>
</select>
Now we can use shift or control to select more than one option from our list.
If we need to, we can also pre-select some options:
<option value="dority" selected>Dan Dority</option>
What do we do with all this important information once we've collected it? We will obviously need to submit it to the National Clearinghouse for Deadwood Watcher Data (NCDWD) for their ongoing monitoring operations!
<input type="submit" value="submit">
The value attribute here defines what the user sees on the button. You could use "send" or "apple" or whatever, but I like "submit" because it makes it clear that you are bowing down to your overseers.
If you wanted to be fancy, you could also use an image for the button.
<input type="image" name="Freight" src="BlackHillsFreight.jpg" height="200" alt="Black Hills Freight title="Send by Black Hills Freight (submit)">
But what if, in an attempt to complete this survey correctly, we went back and watched all three seasons of Deadwood, and in doing so realized we'd better change several of our answers. We will need to reset the form!
<input type="reset" value="reset">
Again, we could change the value to change the text on the button if we wanted to.
When we've got all our form elements made, we can wrap them up all pretty-like using the fieldset and legend tags.
<form id="deadwood" action="" method="post">
<fieldset>
<legend>
Deadwood Survey
</legend>
*Whole bunch of form stuff here!!
</fieldset>
</form>
